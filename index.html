Đảo ngược Linked List

Dễ Linked List Con trỏ

Cho một danh sách liên kết đơn (Singly Linked List). Hãy viết hàm đảo ngược danh sách đó.
Yêu cầu: Không sử dụng mảng phụ để lưu trữ, độ phức tạp không gian O(1) và thời gian O(N).

Code chuẩn (AI Tutor / Senior) 

/**
 * Đảo ngược Linked List (Chuẩn Clean Code)
 * Time Complexity: O(N)
 * Space Complexity: O(1)
 */
class ListNode {
    int val;
    ListNode next;
    ListNode(int x) { val = x; }
}

class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode curr = head;
        
        while (curr != null) {
            ListNode nextTemp = curr.next; // Lưu trữ node tiếp theo
            curr.next = prev;              // Đảo ngược con trỏ
            prev = curr;                   // Tiến prev lên 1 bước
            curr = nextTemp;               // Tiến curr lên 1 bước
        }
        
        return prev; // prev sẽ là head mới
    }
}

Phương pháp Học tập Lai (Hybrid Learning)

Tình trạng phụ thuộc vào AI để vượt qua các bài tập ngắn hạn là một cái bẫy rất phổ biến từ giai đoạn bùng nổ AI năm 2023. Việc bạn tự đánh giá được điểm yếu về lý thuyết và kỹ năng "thực chiến" là bước quan trọng nhất để định hình lại cách học. Ở năm 2026, một lập trình viên giỏi không phải là người code nhanh hơn AI, mà là người nắm vững nền tảng kiến trúc để có thể kiểm định và định hướng AI một cách chuẩn xác.


Chúng ta hãy cùng thiết lập lại phương pháp học theo mô hình Học tập Lai (Hybrid Learning). Mô hình này giúp bạn vừa lấy lại gốc, vừa nâng cấp kỹ năng điều khiển AI:

1. Nguyên tắc "Hộp cát" (Sandbox Rule) 🏗️

Hãy vạch ra ranh giới rõ ràng giữa việc tự tư duy và việc nhờ AI hỗ trợ.

  • Tư duy không màn hình: Khi gặp một bài toán thuật toán (DSA) hoặc một logic mới, hãy đóng trình duyệt lại. Dùng giấy bút để vẽ sơ đồ khối (flowchart) hoặc viết mã giả (pseudo-code) để giải quyết luồng logic trước.
  • Gõ code "chay": Tắt các công cụ auto-complete mạnh (như GitHub Copilot) khi đang luyện tập nền tảng. Việc tự tay gõ từng dòng code, tự nhớ dấu chấm phẩy hay ngoặc nhọn sẽ giúp bạn xây dựng lại "muscle memory" (ký ức cơ bắp) và sự nhạy bén với lỗi cú pháp.

2. Chuyển AI từ "Người làm thuê" thành "Gia sư" 🧠

Thay đổi hoàn toàn tư duy đặt câu hỏi (prompting) của bạn.

  • Tránh: "Viết cho tôi thuật toán đảo ngược danh sách liên kết." (Cách này giết chết tư duy).
  • Nên dùng: "Tôi đang cố viết thuật toán đảo ngược danh sách liên kết nhưng bị kẹt ở vòng lặp while. Đây là code của tôi. Đừng đưa ra code hoàn chỉnh, hãy chỉ ra tôi đang hiểu sai ở dòng nào và cho tôi 1 gợi ý nhỏ."
  • Dùng AI để kiểm chứng: Sau khi tự viết xong một đoạn code chạy được, hãy gửi nó cho AI và hỏi: "Hãy review đoạn code này như một Senior Developer. Độ phức tạp Big O hiện tại là bao nhiêu? Có cách nào viết theo chuẩn Clean Code hơn không?"

3. Phương pháp "Đọc - Phá - Sửa" (Reverse Engineering) 🔍

Thay vì yêu cầu AI viết toàn bộ dự án, hãy:

  1. Tự viết một tính năng cốt lõi bằng kiến thức của mình.
  2. Yêu cầu AI đưa ra một phiên bản nâng cao hoặc sử dụng design pattern tốt hơn.
  3. Đọc từng dòng code của AI, so sánh với code của bạn và tự tay gõ lại theo cách hiểu mới.
